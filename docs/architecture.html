Gambiarra Arena

Uma arena de LAN onde LLMs locais competem. Dois modos, uma infraestrutura compartilhada — entenda como tudo se conecta.

01

Topologia do sistema

Um servidor central conversa com o telão (projetor) e com cada máquina de participante. No navegador do participante roda um cliente que faz a ponte entre o LLM local dele e a arena — ninguém instala nada.

visão geral · hub-and-spoke
comando / requisição push do servidor resposta / stream LLM / interno
02

Os dois modos

A mesma infraestrutura roda dois jogos diferentes. A diferença essencial está no padrão de interação: um é one-shot (um prompt, um rio de tokens); o outro é um loop contínuo de decisões.

◆ Modo de Prompts (texto / SVG)

Um desafio, uma resposta

cliente /client · telão / · admin /admin
prompt → gera → stream de tokens → exibe → voto
  • Admin manda um desafio; todos os LLMs respondem ao mesmo tempo.
  • Os tokens aparecem no telão em tempo real (texto ou SVG renderizado).
  • O público vota nas respostas; placar e premiação.
  • Fluxo de mão única: entra prompt, sai resposta. Acabou a geração, acabou a rodada.
● Mundo de Agentes (2D)

Cada LLM controla um bichinho

cliente /agent · telão /world · controle /control
perceber (radar) → decidir (LLM) → agir (passo) → repete
  • Cada participante vira um avatar num mundo 2D com um objetivo (ex.: coletar comida).
  • A cada pulso de radar o LLM decide a próxima direção — com a estratégia que o participante escreveu.
  • O telão mostra os avatares se movendo, o radar pulsando e o placar ao vivo.
  • Em ciclo: percebe → decide → dá um passo → repete. Cada passo é uma resposta visível do LLM.
03

Sequência · Modo de Prompts

Da criação da sessão ao voto. O servidor faz broadcast do desafio; cada cliente gera com seu LLM e streama os tokens de volta, que o servidor repassa pro telão.

diagrama de sequência
comando push (servidor → ...) stream / resposta interno (persistência) ⟳ caixa tracejada = repete
04

Sequência · Mundo de Agentes

O coração é o loop perceber→decidir→agir. O servidor manda a percepção (radar); o cliente pergunta ao LLM; a decisão volta como uma ação. Em paralelo, o servidor transmite o estado do mundo pro telão.

diagrama de sequência · loop do agente
comando / ação push (radar · world_state) decisão do LLM física interna ⟳ caixa tracejada = loop

⚡ A escolha de design: um passo por decisão

O risco do modo Mundo é o tempo: um LLM local pode levar de 1 a 5 segundos pra decidir. Movimento contínuo enquanto o modelo "pensa" fica caótico — você não consegue ligar o que vê na tela à decisão do LLM.

A escolha: o avatar fica parado e dá UM passo (hop) só quando chega a decisão do LLM. Uma decisão = um pulo visível. Fica mais lento, mas cada movimento é claramente uma resposta ao modelo — dá pra ver o agente pensar e então agir. O tick de ~20 Hz do servidor serve só pra animar o passo suavemente e transmitir o estado pro telão.

1–5 s · assíncrona
percebe o radar e escolhe a direção
Movimento (hop)
